# 부하 테스트 사용 설명서

## 개요

`load-test.html`을 브라우저에서 열면 두 가지 테스트를 바로 실행할 수 있습니다.

| 탭 | 목적 |
|----|------|
| **동시 발급 테스트** | N명이 동시에 요청 → 선착순 정합성 검증 |
| **단일 유저 중복 요청** | 1명이 N번 동시 요청 → 중복 발급 방지 검증 |

브라우저의 `Promise.all()`로 N개의 HTTP 요청을 동시에 발사하고

성공/실패 수와 Redis 남은 재고를 실시간으로 확인합니다.

---

## 동작 원리

```
브라우저
  └─ Promise.all([fetch, fetch, fetch, ...])  ← N개 동시 발사
          │
          ▼
  Spring Boot API  (POST /api/coupons/{id}/issue)
          │
          ├─ Redis SISMEMBER  → 중복 체크
          ├─ Redis DECR       → 재고 차감 (원자적)
          ├─ Redis SADD       → 사용자 등록
          └─ Kafka Produce    → 비동기 DB 저장
          │
          ▼
  브라우저에 성공/실패 응답 반환
```

> **참고**: 브라우저는 도메인당 최대 6개의 TCP 연결을 사용하므로 100개 요청이 일부 직렬화됩니다.
> 완전한 동시성 테스트는 JUnit `ExecutorService` 또는 `k6`, `wrk` 같은 도구를 사용하세요.

---

## 공통 설정

### 서버 URL

페이지 상단 **서버 URL** 입력란에 실행 중인 서버 주소를 입력합니다.

```
http://localhost:8080
```

두 탭이 이 값을 공유합니다.

미 입력 시 기본적으로 `http://localhost:8080`로 호출이 됩니다.

---

## Tab 1 — 동시 발급 테스트

**"N명이 동시에 요청하면 쿠폰 수량만큼만 발급되는가?"** 를 검증합니다.

### 파라미터

| 필드 | 설명 | 예시 |
|------|------|------|
| **쿠폰 ID** | 테스트할 쿠폰의 ID | `1` |
| **동시 요청 수** | 동시에 요청할 가상 사용자 수 | `200` |

> User ID는 매 테스트마다 1 ~ 1,000,000 범위에서 **중복 없이 랜덤 생성**됩니다.

### 버튼

| 버튼 | 동작 |
|------|------|
| **테스트 시작** | N개 요청 동시 발사 |
| **초기화** | 결과 카운터 및 로그 초기화 |
| **재고 초기화** | DB + Redis 완전 초기화 (아래 참고) |
| **재고 확인** | Redis 현재 재고 조회 |

### 테스트 순서

1. 쿠폰 생성

2. 재고 초기화

3. 테스트 실행

### 결과 해석

| 쿠폰 수량 | 동시 요청 | 기대 결과 |
|---------|---------|---------|
| 100개 | 200명 | 성공 100 / 실패 100 |
| 100개 | 50명 | 성공 50 / 실패 0 |
| 100개 | 100명 | 성공 100 / 실패 0 |

```
발급 성공 수 = 쿠폰 수량  ✅ 정합성 보장
남은 재고   = 0        ✅ 정확한 재고 차감
```

- **성공 수 > 쿠폰 수량**: 동시성 제어 실패 (초과 발급)
- **성공 수 < 쿠폰 수량**: 의도치 않은 롤백 또는 재고 설정 오류

## Tab 2 — 단일 유저 중복 요청 테스트

**"같은 사람이 동시에 여러 번 요청해도 1번만 발급되는가?"** 를 검증합니다.

### 파라미터

| 필드 | 설명 | 예시 |
|------|------|------|
| **쿠폰 ID** | 테스트할 쿠폰의 ID | `1` |
| **User ID** | 중복 요청을 보낼 사용자 ID | `1` |
| **요청 횟수** | 동시에 보낼 요청 수 | `10` |

### 기대 결과

```
발급 성공: 1
발급 실패: N-1  (중복 발급 차단)
남은 재고: 쿠폰 총량 - 1
```

### 테스트 순서

1. **[재고 초기화]** 클릭 → 이전 발급 이력 제거
2. User ID 입력 (예: `1`)
3. 요청 횟수 입력 (예: `10`)
4. **[테스트 시작]** 클릭

## 참고: 실제 부하 테스트 도구

브라우저 테스트는 TCP 연결 수 제한으로 완전한 동시성을 재현하기 어렵습니다.
정밀한 성능 측정이 필요하다면 아래 도구를 사용하세요.

```bash
# k6 (권장)
k6 run --vus 200 --duration 10s script.js

# Apache Bench
ab -n 1000 -c 200 -p body.json -T application/json \
   http://localhost:8080/api/coupons/1/issue
```
